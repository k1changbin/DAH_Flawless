# DAH 2026 프로젝트 — 쉬운 설명서 (팀원용)

> 이 문서는 `DAH2026_MASTER_PLANNING.md`(진짜 설계 문서)를 **고등학생도 이해할 수 있게** 풀어 쓴 버전이다.
> 마스터플래닝이 너무 빽빽하면 이걸 먼저 읽고, 용어가 막히면 맨 뒤 **용어 사전**을 보면 된다.

> 용어 업데이트(2026-07-04): 이 문서의 옛 `world` 표현은 현재 구현 기준으로 scorer가 보는 정답지인 `scorer_truth(state["world"])`에 해당한다. 현실 원천 신호는 `raw_world`, AI가 받은 입력은 `blue_observed`라고 구분한다.

---

## 1. 우리가 만드는 게 뭐야? (한 문장)

> **"드론을 조종하는 AI한테 거짓 정보를 흘리는 해커(Red)와, 그 거짓을 잡아내는 경비팀(Blue)이 한 판씩 붙는 컴퓨터 게임을 만든다."**

진짜 드론을 해킹하는 게 아니라, **컴퓨터 안에서 돌아가는 시뮬레이션(가상 실험)**이다. 진짜 무기도, 진짜 통신도 안 건드린다. 오직 "AI가 보는 정보(데이터)"만 공격하고 방어한다.

---

## 2. 등장인물

| 누구 | 역할 | 쉽게 말하면 |
|---|---|---|
| **관제 AI** | 드론한테 명령 내리는 똑똑이 | 근데 **남이 주는 정보를 믿고** 판단함 |
| **Red (공격자)** | 그 똑똑이한테 거짓 정보를 흘림 | 사기꾼. "배터리 80%야~"(사실 20%) |
| **Blue (경비팀 4명)** | 거짓을 잡고 막고 복구함 | 4명이 역할 나눠서 일함 |
| **심판 (scorer)** | 누가 이겼는지 판정 | 진실을 아는 유일한 존재 |
| **진행자 (Orchestrator)** | 게임을 라운드별로 돌림 | 게임 사회자. AI가 아니라 그냥 진행 코드 |

### Blue 경비팀 4명이 하는 일
1. **탐지반** — "어? 이 정보 이상한데?" 찾아냄
2. **임무반** — "이거 무시하면 드론 추락함? 그냥 넘어가도 됨?" 위험도 판단
3. **방어반** — "그럼 이 정보 잠깐 막자" 실제로 막음
4. **보고반** — "사람아, 방금 이런 일 있었어" 보고서 작성

## 3. 게임은 어떻게 진행돼? (라운드제)

가위바위보처럼 **한 판(라운드)씩** 돈다.

```
1라운드: Red가 거짓 흘림 → Blue가 잡거나 못 잡음 → 점수 기록
2라운드: 또 흘림 → 또 막음 → 점수
...
10라운드까지 반복
```

매 라운드마다 **누가 이겼는지(Red/Blue)** 점수표에 적고, 모든 과정을 기록 파일로 남긴다.

## 4. 가장 중요한 개념 5개 (비유로)

### ① 진짜 상태 vs AI가 보는 상태 (world vs observed)

이게 우리 프로젝트의 **심장**이다.

- **world(월드)** = 진짜 상태. 예: 진짜 배터리는 20%, 모터는 고장.
- **observed(옵저브드)** = AI가 화면으로 보는 상태. 예: 배터리 82%, 모터 정상. 여기서 82%는 큰 차이를 보여주는 `loud_demo` 예시다.

**Red의 공격 = 이 둘을 다르게 벌려놓는 것.** 진짜는 20%인데 AI 화면엔 82%로 보이게 조작할 수 있다. 다만 기본 훈련에서는 이보다 작은 `aggressive` profile 값을 쓴다.
**Blue의 탐지 = AI 화면(observed)만 보고 "어 이거 이상한데?"를 눈치채는 것.**

> 비유: 시험 채점할 때 **정답지(scorer_truth, 코드에서는 state["world"])**와 **학생 답안(observed)**이 있다고 생각해. Red는 답안을 몰래 고치는 사람, Blue는 답안만 보고 "이거 고친 흔적 있는데?"를 찾는 사람.

### ② 진실 분리 (Blue는 정답지를 못 본다)

**Blue 경비팀은 scorer_truth(정답지)를 절대 못 본다.** 만약 본다면? "정답 보고 채점하는" 거라 사기지. 게임이 무의미해진다.

그래서 Blue한테 정보를 줄 때 **state["world"] 키를 지우고(가리고) 준다.** 이 "지우는 작업"이 바로 **redact_state(리댁트)** 다. (→ 용어 사전 참고)

심판(scorer)만 진짜 정답을 알고, "Red가 진짜로 속였는지 / Blue가 잘 잡았는지"를 판정한다.

> 비유: 형사(Blue)는 범인이 누군지 처음부터 알면 안 됨. 단서(observed)만 보고 추리해야 함. 정답은 판사(scorer)만 갖고 있음.

### ③ 불변식 (Blue가 "그냥 if문"이 아닌 이유)

Blue는 "이건 배터리 공격이다!"를 미리 알고 막는 게 **아니다.** 대신 **"말이 안 되는 조합"**을 찾는다.

예: "`loud_demo`에서는 배터리가 82%라는데, 배터리 닳는 속도는 비정상적으로 빠르네? → 둘이 모순이잖아. 수상해."

이렇게 **"원래는 절대 안 깨져야 하는 규칙(불변식)"**이 깨지면 공격으로 의심한다. 공격 이름을 몰라도 잡을 수 있다.

> 비유: 누가 "나 지금 서울에 있어"라는데 사진 배경이 부산 바다면? 어떤 거짓말인지 몰라도 "뭔가 안 맞네"를 알 수 있잖아. 그게 불변식.

### ④ 신뢰 예산 & 피로스의 승리 (과잉 방어도 지는 거다)

Blue가 의심될 때마다 무조건 다 막으면? 드론이 아무것도 못 하고 임무 실패한다. **막다가 임무를 망치는 것.**

그래서 모든 방어에는 **비용**이 든다. 너무 많이 막으면 **"임무 가용성"**이 바닥나서, Red는 직접 안 뚫고도 이긴다. 이걸 **"소모 승리"** 또는 **"피로스의 승리"**라고 부른다.

> 비유: 감기 걱정된다고 1년 내내 집에만 있으면 감기는 안 걸리지만 인생이 망함. "안전"만 챙기다 "삶(임무)"을 잃는 거. 방어도 똑같다.

핵심 한마디: **"다 막아도 임무가 망가지면 진 거다."**

### ⑤ 공진화 (둘 다 점점 똑똑해진다)

Red와 Blue가 **서로를 보고 학습**한다.

- Red: "이 공격은 자꾸 막히네 → 덜 쓰자. 저 공격은 잘 통하네 → 더 쓰자."
- Blue: "이건 자꾸 헛다리 짚네 → 둔감하게. 저건 자꾸 놓치네 → 예민하게."

라운드가 갈수록 둘 다 진화해서, **"진짜로 싸우는 것처럼" 보이는 그래프**가 나온다. 이게 우리 발표의 핵심 그림.

> 비유: 가위바위보를 여러 판 하면서 상대 버릇을 읽고 전략을 바꾸는 거랑 같음. 둘 다 적응함.

## 5. 우리가 "안 하는" 것 (중요)

- ❌ 진짜 드론 해킹
- ❌ 진짜 전파 방해(재밍)
- ❌ 진짜 통신망 침투

우리는 **"AI가 믿는 정보"만 공격하는 안전한 컴퓨터 실험**이다. 이걸 보고서에 **정직하게** 쓰는 게 오히려 점수가 된다. ("우리는 이건 일부러 범위 밖에 뒀습니다"가 성숙해 보임)

## 6. 각자 뭘 만들어? (역할)

| 팀원 | 맡는 부분 | 쉽게 |
|---|---|---|
| A | Red(공격자) + 공격 종류들 | 사기꾼 AI 만들기 |
| B | Blue 탐지·임무 판단 | "이상한데?" 잡는 부분 |
| C | 진행 코드 + 방어 처리 + 도커 | 게임 엔진 + 실행환경 |
| D | 보고서 생성 + 그림/대시보드 + 문서 | 결과 보여주기 + 보고서 |

## 7. 공격은 3종만 깊게 만든다

공격 종류를 많이 만들기보다, **3개를 확실하게** 만들어서 로그로 증명한다.

| 공격 이름 | 뭐 하는 공격 |
|---|---|
| **PRIORITY_POISONING** | 임무 우선순위 점수를 조작 → AI가 엉뚱한 구역을 중요하다고 착각 |
| **TELEMETRY_FDI** | 배터리·모터 상태를 거짓으로 → AI가 "정상 비행 가능"으로 오판 |
| **TIME_DESYNC_REPLAY** | 명령의 시간/순서를 조작 → 옛날 명령을 최신인 것처럼 처리 |

(나머지 공격들은 "목록엔 있지만 거의 안 나오게" 설정만 해둔다.)

## 8. 발표/보고서엔 뭐가 들어가? (시각화)

예선은 **PDF 보고서 심사**다. 그래서 시각화를 **두 층**으로 나눠야 한다. (중요!)

### 층 1. 보고서에 박는 정적 그림 = 진짜 점수 (Must)

심사위원은 PDF만 본다. **라이브 대시보드를 켜보지 않는다.** 그래서 보고서에 들어가는 **그림 이미지(PNG)가 곧 점수**다. 이건 꼭 만들어야 한다.

꼭 들어갈 그림:
- 협력 구조도 (Red/Blue/심판 + "Blue는 정답 못 봄" 표시)
- 진짜 상태 vs AI가 본 상태 비교표 (조작된 칸 빨강)
- 공격 흐름도 (상황→조작→오판→피해)
- 탐지·격리·복구 표
- 점수표 (라운드별 승자)
- (여유되면) 공진화 그래프, 가용성 하락 곡선

현재 repo에는 보고서 그림 생성물과 PDF/ZIP 생성 스크립트를 넣지 않는다. 그림은 JSONL 로그, Streamlit 화면, 또는 별도 그림 도구를 바탕으로 보고서 작성 단계에서 따로 만든다.

### 층 2. 라이브 대시보드 + 데모 영상 = 보조재 (Nice)

- **Streamlit 대시보드**: 실행하면 화면에서 실시간으로 보이는 거. 멋지지만 **예선 필수는 아님.**
- **데모 영상**: "프로그램 돌리는 화면"을 녹화하는 거. 설득력은 있지만 **점수 본체는 아님.**

> 한마디: **보고서 그림(Must)이 먼저, 대시보드·영상(Nice)은 나중.** 시간이 없으면 대시보드는 버려도 되지만, 보고서 그림은 버리면 안 된다.

## 9. 용어 사전 (어려운 말 풀이)

개발하다 보면 자꾸 나오는 단어들. 막히면 여기 보면 된다.

| 용어 | 읽는 법 | 무슨 뜻이야? |
|---|---|---|
| **schema (스키마)** | 스키마 | "데이터의 정해진 모양/틀". 예: 학생부 양식처럼 "상태 정보는 이런 칸들로 이루어진다"를 미리 정해둔 것. `schemas.py` = 그 양식들을 모아둔 파일. |
| **state (스테이트)** | 스테이트 | "현재 상태"를 담은 데이터 덩어리. 배터리·위치·통신상태 등이 다 들어있는 한 묶음. 게임은 이 state를 라운드마다 변형시키는 것. |
| **scorer_truth / observed** | 스코어러 트루스 / 옵저브드 | scorer_truth=채점용 정답지(코드 키는 state["world"]), observed=AI가 보는 상태(답안지). 공격은 이 둘을 다르게 만드는 것. |
| **raw_world** | 로 월드 | 현실 원천 신호. RF/GNSS/SATCOM/C2 방출처럼 아직 AI 입력으로 정리되기 전의 데이터. |
| **redact_state (리댁트 스테이트)** | 리댁트 | "가린다"는 뜻. 문서에서 비밀 부분을 검은 펜으로 칠하는 거(▒▒▒) 본 적 있지? Blue한테 정보 줄 때 scorer_truth 정답지를 지워서 주는 함수. |
| **agent (에이전트)** | 에이전트 | "스스로 판단해서 행동하는 AI 일꾼". Red도 agent, Blue 4명도 각각 agent. 사람이 일일이 안 시켜도 알아서 관찰→판단→행동함. |
| **invariant (인베리언트)** | 인베리언트 | "절대 안 깨져야 하는 규칙". 예: "배터리가 높으면 닳는 속도가 빠를 리 없다". 이게 깨지면 공격 의심. |
| **catalog (카탈로그)** | 카탈로그 | "공격 목록표". 어떤 공격들이 있고 각각 얼마나 자주 나올지(weight)를 적어둔 것. |
| **weight (웨이트)** | 웨이트 | "가중치 = 뽑힐 확률". 숫자가 클수록 그 공격이 자주 나옴. 0이면 안 나옴. |
| **situation_tags (시추에이션 태그)** | 태그 | "지금 상황 라벨". 예: "GPS 의존 중", "통신 약함". Red가 이걸 보고 제일 잘 먹힐 공격을 고름. |
| **defense queue (디펜스 큐)** | 큐 | "방어 대기줄". 방어는 즉시 안 되고 줄 서서 처리됨. 동시에 2개만 가능하면 나머진 대기. (큐=줄서기) |
| **scorer (스코어러)** | 스코어러 | "심판/채점기". 진짜 상태를 아는 유일한 존재. 누가 이겼는지 규칙대로 판정. |
| **JSONL (제이슨엘)** | 제이슨엘 | 기록 파일 형식. 한 줄에 한 라운드씩 차곡차곡 쌓는 로그. 엑셀 행처럼 한 줄 = 한 사건. |
| **seed (시드)** | 시드 | "랜덤 고정 번호". 같은 seed를 넣으면 매번 똑같은 결과가 나옴. → 심사위원이 "우리도 돌려봤더니 똑같네" 재현 가능. |
| **hash chain (해시 체인)** | 해시 체인 | "변조 방지 사슬". 기록끼리 서로 연결해서, 한 줄이라도 몰래 고치면 사슬이 끊겨 들통남. 블록체인이랑 비슷한 원리. |
| **E2E (엔드 투 엔드)** | 이투이 | "처음부터 끝까지 한 번 관통". 공격→탐지→방어→기록까지 한 바퀴가 다 돌아가는 것. |
| **MVP** | 엠브이피 | "최소 기능 제품". 일단 핵심만 돌아가게 만든 첫 버전. 완벽보다 "돌아가는 것" 먼저. |
| **Docker (도커)** | 도커 | "어디서나 똑같이 실행되게 포장하는 도구". 내 컴퓨터에서 되던 게 심사위원 컴퓨터에서도 똑같이 돌게 해줌. |
| **Orchestrator (오케스트레이터)** | 오케스트레이터 | "전체 진행자". 라운드를 순서대로 돌리고 각 AI를 호출하는 중심 코드. (지휘자처럼) |
| **co-evolution (코에볼루션)** | 공진화 | Red와 Blue가 서로 보고 같이 진화하는 것. |
| **availability (어베일러빌리티)** | 가용성 | "임무를 수행할 수 있는 여력". 너무 많이 방어하면 이게 깎여서 임무 실패. |
| **feasibility (피저빌리티)** | 실현가능성 | "이 공격이 현실에서 진짜 가능한가" 등급. real(가능) / abstracted(추상화) / out_of_scope(범위밖). |
| **matplotlib (맷플롯립)** | 맷플롯립 | 파이썬으로 그래프·차트를 **이미지(PNG)로 그려주는 도구**. 보고서에 박을 그림을 이걸로 뽑는다. |
| **PNG (피엔지)** | 피엔지 | 그림 이미지 파일 형식. 보고서 PDF에 붙이는 정적(움직이지 않는) 그림. |
| **report figure (보고서 그림)** | 리포트 피겨 | JSONL 로그, Streamlit 화면, 또는 별도 그림 도구로 만든 보고서용 정적 그림. 현재 repo에는 생성물을 고정 보관하지 않는다. |
| **Streamlit (스트림릿)** | 스트림릿 | 파이썬으로 **웹 대시보드 화면**을 쉽게 만드는 도구. 실시간으로 보여줄 수 있지만 예선 필수는 아님(Nice). |
| **Must / Should / Nice** | 머스트/슈드/나이스 | 할 일 우선순위. Must=꼭(코어), Should=하면 가점, Nice=여유되면. 시간 없으면 Nice부터 버린다. |
