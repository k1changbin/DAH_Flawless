# DAH 대시보드 전면 리디자인 구상서 (FRONTEND_DESIGN_SPEC)

> 작성: 2026-07-08, Fable 5 (구상 전담). 구현은 이 문서를 받은 하위 모델이 수행한다.
> 이 문서가 유일한 진실이다. 구현 중 이 문서와 충돌하는 판단이 필요하면 임의로 결정하지 말고 사용자에게 질문할 것.

## 0. 디자인 리드 (Design Read)

**"군사 작전 관제(C2) HUD 언어를 쓰는 사이버전 리플레이 대시보드. 청중은 해커톤 심사위원과 일반 관람자. 다크 네이비 기반 + 시안 HUD 스트로크 + Red/Blue 진영 대비, 중앙 3D 전장 씬을 2.5D HUD 패널들이 둘러싸는 구성."**

다이얼 (taste-skill 기준):
- `DESIGN_VARIANCE: 7` — 비대칭 패널 배치, 창-밖-창 오프셋. 단 데이터 정렬은 그리드 엄수.
- `MOTION_INTENSITY: 7` — 부팅 시퀀스·타임라인 재생·공격 펄스는 화려하게, 정보 패널은 절제.
- `VISUAL_DENSITY: 7` — 콕핏. 모든 숫자는 mono 폰트. 카드 박스 남발 금지, 1px 라인과 여백으로 구분.

**이 프로젝트는 대시보드이므로 taste-skill의 랜딩페이지 전용 규칙(히어로/CTA/로고월 등)은 적용하지 않는다.**
적용하는 것: 안티슬롭 금지 목록(9장), 색·타이포 규율(4.1/4.2), 모션 동기 원칙(5장), 다크모드 프로토콜(8장), em-dash 금지.

## 2. 기술 스택 (확정)

- **React 18 + Vite + TypeScript**
- **3D**: `three` + `@react-three/fiber` + `@react-three/drei` — 중앙 씬 전용 (하이브리드 원칙, 4장)
- **모션**: `motion` (`import { motion } from "motion/react"`) — UI 패널 전부. **Three와 Motion을 같은 컴포넌트 트리에서 섞지 말 것.** 3D 씬은 R3F의 `useFrame`으로만 애니메이션.
- **상태**: `zustand` 단일 스토어 (현재 스텝, 포커스 진영, 재생 상태, 무결이 상태)
- **스타일**: Tailwind v4 + CSS 변수 토큰 (아래 3장)
- **아이콘**: `@phosphor-icons/react` 한 가족만, `strokeWidth 1.5` 통일. 손그림 SVG 아이콘 금지.
- **제출 빌드**: `vite-plugin-singlefile` — 단일 HTML 산출로 기존 "정적 제출" 방향 유지. combat_replay.json은 빌드 시 인라인.
- **데이터**: 기존 `build_frontend_combat_log` → `combat_replay.json` projection을 그대로 소비. **백엔드/스키마 수정 금지.**
- 새 앱 위치: `frontend/` 디렉터리 (기존 `streamlit_app.py`, `scripts/build_static_site.py`는 삭제하지 말고 그대로 둘 것).

## 3. 디자인 토큰

### 3.1 컬러 (CSS 변수, 다크 단일 테마 고정 — 관제 대시보드이므로 라이트 모드 없음. 페이지 테마 락)

```css
:root {
  /* 표면 — 순수 검정 금지 */
  --surface-0: #0A0F1C;   /* 페이지 배경, 잉크 네이비 */
  --surface-1: #0F1626;   /* 패널 배경 */
  --surface-2: #16203A;   /* 패널 내 elevated 요소 */
  --line-hud:  #2A3A5C;   /* HUD 프레임 기본 스트로크 */
  --line-hud-active: #5FD4F5; /* 포커스/활성 프레임 스트로크 (시안) */

  /* 진영 — 유일한 의미 색. 다른 용도로 쓰지 말 것 */
  --red-ops:   #FF5D45;   /* Red 공격 (웜 시그널 레드) */
  --red-dim:   #7A3328;
  --blue-def:  #4FA3FF;   /* Blue 방어 (스틸 블루) */
  --blue-dim:  #27476B;

  /* 텍스트 */
  --text-hi:   #E8EEF7;
  --text-mid:  #9FB0C9;
  --text-low:  #5A6B85;

  /* 판정 (ZTA 6결정·승패) */
  --ok:        #58D6A3;   /* ALLOW, 승리 */
  --warn:      #F5C542;   /* 조건부 판정 */
  --deny:      #FF5D45;   /* DENY — red-ops와 동일값 공유 허용 */

  /* 무결이 전용 — 대시보드 어디에도 재사용 금지 */
  --mugyeol-a: #C77DFF;
  --mugyeol-b: #6EE7F0;
  --mugyeol-c: #FF8FD8;
}
```

규율: 액센트 남발 금지. 시안(`--line-hud-active`)은 "현재 포커스"에만, red/blue는 진영 데이터에만, 무결이 그라데이션은 무결이에게만. 보라/네온 글로우를 일반 UI에 쓰지 말 것(무결이는 예외, 의도된 대비). AI-퍼플 그라데이션 배경 금지.

### 3.2 타이포그래피 (전부 셀프호스트, `font-display: swap`)

- **한글 본문/UI**: `Pretendard Variable`
- **HUD 디스플레이(라틴 대문자 라벨, 패널 타이틀)**: `Chakra Petch` (600/700)
- **숫자·좌표·타임코드 전부**: `JetBrains Mono` — VISUAL_DENSITY 7 콕핏 규칙, 예외 없음
- Inter/Roboto/Arial 금지. 크기 위계: 패널 타이틀 12px 대문자 letter-spacing 0.08em / 본문 13-14px / 핵심 수치 20-28px mono.

### 3.3 형태 시스템 (Shape Lock)

- **패널 프레임**: radius 0 + 코너컷(clip-path로 모서리 1~2곳 45° 절단, 이미지 #8) + 1px `--line-hud` 스트로크. 절단 코너 근처에 6px 짧은 액센트 세그먼트 1개 허용.
- **패널 내부 요소**(버튼, 인풋, 태그): radius 6 고정.
- **무결이 관련**: 완전 원형/블롭만.
- 이 3규칙 외 radius 값 금지. 문서화된 혼합 시스템이므로 일관 적용.

### 3.4 배경·질감

- 페이지 배경: `--surface-0` + 아주 미세한 원근 그리드(1px 라인, 투명도 4-6%, 소실점 하단 중앙) — 이미지 #9의 지도 그리드 느낌.
- 그레인/노이즈 오버레이는 `fixed inset-0 pointer-events-none` 요소에만, 투명도 3% 이하.
- 패널 배경은 반투명 + `backdrop-filter: blur(12px)` 허용(3D 씬 위에 떠 있으므로 동기 있음). `prefers-reduced-transparency` 시 솔리드 폴백.

## 4. 레이아웃 (창-밖-창 + 듀얼 스위칭)

```
┌────────────────────────────────────────────────────────────────┐
│ 커맨드 바 (h-16): 라운드 선택 · 재생컨트롤 · 스텝 타임코드 · 승패 배지 │
├──────────┬──────────────────────────────────────┬──────────────┤
│          │                                      │              │
│ RED OPS  │        중앙 3D 전장 씬 (R3F)          │  BLUE DEF    │
│ 패널      │   UAV/UGV/SATCOM 노드 + 링크 +        │  패널         │
│ (좌)     │   공격 경로 발광 라인                   │  (우)         │
│          │                                      │              │
├──────────┴──────────────────────────────────────┴──────────────┤
│ 타임라인 레인 (h-28): 스텝 스크러버 + 공격/탐지/ZTA 이벤트 마커      │
└────────────────────────────────────────────────────────────────┘
  ↖ 위성 창(창-밖-창): suspicion 스파크라인 · ZTA 정책 타임라인 ·
     텔레메트리 피드 · 이벤트 로그 — 메인 프레임 모서리에 겹쳐 배치      ↗
                                              (우하단) 무결이 FAB ●
```

### 4.1 창-밖-창 (이미지 #11)

- 메인 3D 씬이 "주 프레임". 위성 패널 4개(suspicion 스파크라인, ZTA 정책 타임라인, 텔레메트리 피드, 이벤트 로그)는 주 프레임의 경계에 **일부 걸쳐서** 배치 — 완전히 안도 밖도 아닌 오프셋(예: `translate(-12px, -12px)`가 프레임 경계를 물게).
- 각 위성 패널은 헤더 드래그로 위치 이동 가능(선택 구현, P5), 더블클릭으로 접기/펼치기.
- 1440px 미만: 위성 패널은 좌우 도킹 컬럼으로 수렴. 1024px 미만: 탭 시트로 수렴(모바일 대응은 데모 특성상 최소한만).

### 4.2 RED/BLUE 듀얼 패널 스위칭 (이미지 #12, 핵심 인터랙션)

- 기본 상태: 좌 RED OPS(w-72), 우 BLUE DEF(w-72) 대칭.
- **RED 패널 클릭 → RED 포커스 모드**: RED가 w-[26rem]으로 확장 + 프레임 스트로크가 `--red-ops`로 점등 + 내부 상세(공격 3종 TIME_DESYNC_REPLAY/TELEMETRY_FDI/PRIORITY_POISONING 타임라인, 대상 자산, 변조 파라미터) 노출. BLUE는 w-14 레일로 축소(아이콘+suspicion 게이지만).
- BLUE 클릭 시 대칭 동작 (ZTA 6결정 스트림, internal/external observe 비교, 격리/복구 액션 로그).
- 전환은 `motion`의 `layout` 애니메이션, `type: "spring", stiffness: 300, damping: 32`. 스위칭 무제한 반복 가능, 300ms 내 완료. `Tab`+`Enter` 키보드 접근 가능, `aria-expanded` 부여.
- 3D 씬 카메라도 포커스 진영 쪽으로 미세하게 도브(lerp 0.6s) — 씬과 패널이 한 몸처럼 반응.

## 5. 중앙 3D 씬 (하이브리드 원칙)

**WebGL은 이 씬 하나뿐이다. 다른 곳에 Canvas 추가 금지.**

- **구성**: 낮은 채도의 와이어프레임 지형 평면(어두운 청록, 이미지 #6) 위에 UAV(공중, y+), UGV(지상), SATCOM(최상단) 노드 배치. 노드는 단순 기하(octahedron/box/icosahedron) + 진영 무관 중립색, 상태에 따라 emissive 변화.
- **링크**: 노드 간 통신 링크는 얇은 라인. **공격 발생 스텝에서는 red 발광 튜브 라인이 경로를 따라 흐르고**(이미지 #4의 3D 경로 라인), 탐지/차단 스텝에서는 blue 펄스가 역방향으로 흐른다. `TubeGeometry` + 텍스처 오프셋 스크롤 또는 shader 대신 drei `Line`+dash 오프셋으로 구현(단순 우선).
- **카메라**: 기본 3/4 부감, 유저 드래그 orbit 허용(damping), 자동 회전 없음(현기증 방지). 스텝 이벤트 시 해당 노드로 부드러운 focus lerp.
- **성능 가드**: 노드 수 ≤ 12, 라이트 2개(ambient+directional), 그림자 끄기, `dpr={[1, 1.5]}`, 포스트프로세싱은 Bloom 하나만 낮은 강도로(발광 라인용). 60fps 미달 시 Bloom부터 제거.
- **폴백**: WebGL 불가 환경이면 씬 자리에 2.5D SVG 토폴로지 다이어그램(동일 데이터, CSS 글로우) 렌더. `prefers-reduced-motion` 시 라인 흐름 정지, 상태색만 변경.

## 6. 마스코트 "무결이" (음성 어시스턴트)

이름 유래: 무결성(integrity). ZTA 게이트의 얼굴 역할.

### 6.1 비주얼

- 우하단 FAB(56px 원): 내부에 소형 무결이가 떠 있음. 클릭 → 코너컷 미니 창(320×380)이 FAB에서 스프링으로 자라나며 열림(`layoutId` 공유로 FAB→창 morph).
- **본체**: SVG/CSS 블롭. `border-radius: 42% 58% 55% 45% / 48% 44% 56% 52%` 형태의 값 4세트를 8s 루프로 morph(keyframes). 내부 채움은 `--mugyeol-a/b/c` 3색 `conic-gradient` + `filter: blur` 레이어로 은하수/오로라가 안에서 흐르는 느낌(그라데이션 `background-position` 애니메이션). 외곽에 얇은 하이라이트 링.
- 대시보드의 네이비/시안과 의도적으로 충돌하는 유일한 존재. 이 3색은 무결이 밖에서 절대 사용 금지.

### 6.2 상태 머신

| 상태 | 트리거 | 모션 |
|---|---|---|
| idle | 기본 | 3.2s 주기 호흡(scale 1→1.03), 블롭 morph 느리게 |
| listening | 마이크 버튼 클릭, `SpeechRecognition` 활성 | **꿈틀 모드**: morph 주기 8s→1.2s, 진폭 2배, 음성 입력 볼륨(AnalyserNode)에 비례해 scale 튀김 |
| thinking | 음성 파싱 후 명령 매칭 중 | 빠른 회전성 그라데이션 흐름 |
| speaking | 응답 출력(`speechSynthesis`, ko-KR) | 발화 리듬에 맞춰 상하 바운스 |
| error | 인식 실패 | 좌우 shake 1회 + 텍스트 폴백 안내 |

### 6.3 기능 범위 (정직하게 데모 수준)

- Web Speech API `SpeechRecognition`(ko-KR) — **Chrome 계열 전용**. 미지원 브라우저에선 마이크 버튼 숨기고 텍스트 입력만.
- LLM 없음. **사전 정의 명령 매핑**만: "3라운드 보여줘"(라운드 전환), "공격 뷰/방어 뷰"(패널 포커스), "재생/정지", "다음 스텝", "누가 이겼어"(승패 요약 발화), "제타 판정 보여줘"(ZTA 타임라인 열기). 매칭은 키워드 기반, 실패 시 무결이가 가능한 명령 목록을 보여줌.
- 마이크는 명시적 클릭으로만 활성화(자동 청취 금지), 활성 중 시각 표시 필수.

## 7. 모션 스펙 (animate 스킬 기준)

### 7.1 부팅 시퀀스 (페이지 로드, 한 번의 잘 짜인 오케스트레이션)

1. 0ms: 배경 그리드 fade-in (400ms, linear)
2. 150ms: 패널 프레임들이 SVG 스트로크 draw-in (`stroke-dashoffset`, 500ms, ease-out) — HUD 부팅 느낌
3. 400ms: 패널 내용 stagger (컨테이너 `staggerChildren: 0.06`, 각 항목 y:16→0 + opacity, 300ms, `cubic-bezier(.23,1,.32,1)`)
4. 600ms: 3D 씬 opacity 0→1 + 카메라 도입 lerp (800ms)
5. 1200ms: 무결이 FAB pop (spring, scale 0→1)

이후 상시 루프 애니메이션은 무결이 idle과 3D 링크 흐름뿐. 정보 패널에 무한 루프 금지.

### 7.2 공통 규칙

- 등장 ease-out 200-300ms / 퇴장 ease-in 150-200ms (등장의 ~75%) / transform·opacity만 애니메이션.
- 패널 스위칭·모달·무결이 창: spring (`stiffness 300-400, damping 30-32`).
- 타임라인 스크럽 중에는 등장/퇴장 애니메이션 전부 즉시 모드(스크럽이 끊기면 안 됨).
- `prefers-reduced-motion`: 부팅 시퀀스 생략(즉시 표시), 블롭 morph 정지, 3D 라인 흐름 정지. 필수 구현.
- 이벤트 피드백: 공격 스텝 도달 시 해당 위성 패널 프레임이 red로 1회 펄스(600ms), DENY 판정 시 ZTA 레인에 deny 마커 pop. 모션은 전부 "상태 전달" 동기가 있어야 하며 장식용 금지.

## 8. 데이터 계약 (수정 금지, 소비만)

- 입력: `combat_replay.json` — `build_frontend_combat_log`가 생성. 필드: `winner_side`, `winner_detail`, `combat_steps[]`(스텝별 공격/탐지/격리/복구 이벤트), `zero_trust` 섹션(`step_log.zta_decisions`, `entry.zta_policy`, 6결정, `policy_decision_correctness`), suspicion 시계열, `blue_observed`의 internal/external observe.
- **Blue 패널에는 scorer truth를 절대 표시하지 않는다**(프로젝트 핵심 불변식). Blue 쪽 UI는 `blue_observed`만 소비. truth 대비 시각화가 필요하면 "관전자 뷰" 라벨을 명시한 별도 오버레이로만.
- 승패 표시: `winner_side` + `winner_detail` 2단 분해(커맨드 바 배지 + 클릭 시 상세). `PARTIAL_BREACH` 상태 파싱 포함.
- 빌드 파이프라인: `python scripts/generate_frontend_log.py`(기존) → JSON → Vite 빌드에서 인라인. 하위 모델은 JSON 스키마를 먼저 실물로 확인한 후 타입 정의(`types/replay.ts`)를 작성할 것.

## 9. 구현 페이즈 (하위 모델 작업 순서, 각 페이즈 끝날 때마다 스크린샷 검증)

| **P1** | 커맨드 바 + 듀얼 패널 + 스위칭 + 타임라인 레인 (데이터는 실제 JSON) | RED/BLUE 스위칭이 스프링으로 동작, 키보드 접근 가능 |
| **P2** | 중앙 3D 씬 (노드/링크/공격 라인/카메라) + 2D 폴백 | 60fps, 스텝 이동 시 라인 펄스 동작 |
| **P3** | 위성 창 4종 + 창-밖-창 배치 + 스텝 동기화(스크럽→씬+패널 전체 반응) | 스크럽 중 프레임 드랍 없음 |
| **P4** | 무결이 (블롭+상태머신+음성 명령 6종+미니 창) | Chrome에서 음성 명령 동작, 미지원 브라우저 폴백 확인 |
| **P5** | 부팅 시퀀스, 이벤트 펄스, reduced-motion, 성능/접근성 마감 | 아래 10장 체크리스트 전부 통과 |

각 페이즈는 독립 커밋. **P2가 리스크 최대** — 막히면 3D 범위를 줄이되(Bloom 제거 → 라인 흐름 단순화 → 최후에 2.5D 폴백 승격) 다른 페이즈를 볼모로 잡지 말 것.

## 10. 프리플라이트 체크리스트 (P5에서 전수 확인)
